# SQL Analytics Portfolio

Production-style SQL for common analytics problems — window functions, cohort retention, funnel analysis, and data-quality checks. Every query has a written problem statement and approach note in the file.

Runs locally with **DuckDB** — no database or cloud account needed.

## Quick start

```bash
pip install duckdb pandas
python run_queries.py          # run all sections
python run_queries.py window   # just window functions
python run_queries.py cohort   # just cohort retention
python run_queries.py funnel   # just funnel analysis
python run_queries.py quality  # just data-quality checks
```

## Dataset

Synthetic e-commerce data (2024–2025) generated by `datasets/generate.py`:

| Table | Rows | Description |
|---|---|---|
| `orders` | 8 000 | Order-level transactions across 4 regions, 3 channels, 8 products |
| `customers` | 2 000 | Customer dimension with signup date, region, segment |
| `events` | ~16 000 | Clickstream funnel events (page_view → purchase) |

All synthetic, all reproducible (`random.seed(42)`), no employer data.

## Sections

### [01 — Window Functions](01-window-functions/queries.sql)
Running totals, 7-day rolling average, revenue ranking with percentile buckets, day-over-day change with LAG, deduplication with ROW_NUMBER, gaps-and-islands.

### [02 — Cohort Retention](02-cohort-retention/queries.sql)
Monthly signup cohort sizes, customer retention curve (months 0–5), revenue retention by cohort.

### [03 — Funnel Analysis](03-funnel-analysis/queries.sql)
Overall funnel conversion rates with step-over-step drop-off, funnel by channel, median time between steps.

### [04 — Data Quality Checks](04-data-quality-checks/queries.sql)
Null rate per column, duplicate primary key detection, referential integrity, business-rule validation (negative amounts, orders before signup, quantity range), freshness check, row-count reconciliation.

## Dialect

Written for DuckDB / Snowflake syntax. Notes in each file call out where standard SQL differs.
